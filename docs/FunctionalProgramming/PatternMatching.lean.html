The bcount function counts the number of elements in a list that satisfy the given predicate p. The predicate’s codomain is Bool. As a general rule, we will use type Bool, of Booleans, within programs and use the type Prop, of propositions, when stating properties of programs.

The connectives are called or (infix: ||), and (infix: &&), and not (¬) or (!).

This works only for propositions that are decidable (i.e., executable). This is the case for ≤: Given concrete values for the arguments, such as 35 and 49, Lean can reduce 35 ≤ 49 to true. Lean keeps track of decidability using a mechanism called type classes, which will be explained below.
